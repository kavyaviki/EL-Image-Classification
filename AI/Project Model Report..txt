-----Solar Images-----

1. Project Overview

Domain : Solar Energy – Photovoltaic (PV) Module Inspection.
Task: Automated defect detection in Electroluminescence (EL) images of solar cells using Deep Learning.
Categories: Binary Classification (Classes: `defect` and `good`).

2. Technical Stack

Deep Learning Framework: PyTorch
Model Library: `timm` (PyTorch Image Models)
Computational Hardware: NVIDIA GPU (CUDA) used for training acceleration.
Supporting Libraries: NumPy, Matplotlib (Visualization), Scikit-learn (Evaluation Metrics), and tqdm (Progress tracking).

3. Image Pre-processing & Augmentation

To improve model robustness and generalize better to new images, the following pipeline was used:
Input Size: Images are resized to 384 x 384 pixels.
Normalization: Applied using ImageNet statistics:
Mean: `[0.485, 0.456, 0.406]`
Std Dev: `[0.229, 0.224, 0.225]`

Augmentation Techniques (Training Only):
Random Horizontal Flip.
Random Rotation (up to 10 degrees).
Color Jitter (Brightness and Contrast adjusted by 0.2).



4. Training Hyperparameters

These parameters define the training regimen found in the "developed code":
 Batch Size: 16
 Epochs: 40
 Learning Rate (LR): 1e-4 (0.0001)
 Optimizer: Adam Optimizer (`optim.Adam`).
 Loss Function: Cross-Entropy Loss (`nn.CrossEntropyLoss`).

5. Dataset Structure

The project utilizes a standard directory-based split:
 Path: `Dataset_Split/`
 Subfolders: `train`, `val`, and `test`.
 Loader: `DataLoader` with 4 worker threads for efficient data streaming.

4. Confusion Matrix
This is the most important slide for a technical audience. It breaks down exactly where the model gets confused.
 True Positives (TP): Defective cells correctly identified.
 True Negatives (TN): Good cells correctly identified.
 False Positives (FP): Good cells incorrectly flagged as defective (increases inspection costs).
 False Negatives (FN): Defective cells missed (reduces panel quality).
